# AstrBot 车站查询插件

一个用于查询本地数据库车站信息的 AstrBot 插件。

## 功能特性

**车站查询**：通过输入车站名称查询车站详细信息
**车站列表**：查看所有已收录的车站
**添加车站**：管理员可添加新的车站信息
**JSON 数据库**：使用 JSON 格式存储车站数据，方便修改和维护
**多指令支持**：支持多种指令格式，适应不同用户习惯
（数据库内置大部分广州地铁车站以及极小部分上海、北京地铁车站数据，如您想帮助我们完善请提交pr）

## 安装方法

1. 将插件目录 `astrbot_plugin_stationsearcher` 复制到 AstrBot 的 `plugins` 目录下
2. 重启 AstrBot 或在 WebUI 中重载插件
3. 插件将自动创建必要的数据目录和初始数据库

## 使用说明

### 基础命令

- **查询车站**：
  - 输入 `station <车站名>` 或 `/station <车站名>`
  - 例如：`station 北京西站`、`/station 上海虹桥`

- **查看车站列表**：
  - 输入 `station.list` 或 `车站列表`
  - 系统将按城市分组显示所有车站

- **查看帮助**：
  - 输入 `station.help` 或 `车站帮助`
  - 显示详细的使用说明

### 管理员命令

- **添加车站**：
  - 输入 `station.add {"name": "车站名", "city": "城市", "lines": ["线路1", "线路2"]}`
  - 例如：`station.add {"name": "杭州东站", "city": "杭州", "lines": ["地铁1号线", "高铁"]}`
  - 支持添加额外字段如 `address`（地址）和 `description`（简介）

## 数据结构

车站数据库使用 JSON 格式存储，位于 `data/plugin_data/astrbot_plugin_stationsearcher/stations_database.json`。

### 数据格式

```json
{
  "stations": [
    {
      "id": "station001",
      "name": "博丽神社",
      "city": "幻想乡",
      "lines": ["地铁东西线", "地铁南北线", "新干线"],
      "address": "极东之地",
      "description": "博丽巫女的家。"
    }
  ]
}
```

### 字段说明

- `id`：车站唯一标识符（自动生成）
- `name`：车站名称（必填）
- `city`：所属城市（必填）
- `lines`：接入线路列表（必填）
- `address`：车站地址（可选）
- `description`：车站简介（可选）

## 自定义配置

### 修改数据库

您可以直接编辑 `data/plugin_data/astrbot_plugin_stationsearcher/stations_database.json` 文件来修改车站数据。修改后需要重启插件或重载 AstrBot 以应用更改。

### 添加更多字段

如果需要添加更多自定义字段，可以直接在 JSON 文件中为每个车站添加新的键值。插件会自动忽略未知字段，不会影响正常功能。

## 版本要求

- Python >= 3.8.0

## 故障排除

### 常见问题

1. **查询不到车站**
   - 检查输入的车站名称是否正确
   - 确认该车站已添加到数据库中

2. **添加车站失败**
   - 检查 JSON 格式是否正确
   - 确保包含必要的字段（name、city、lines）
   - 验证您是否有管理员权限

## 开发说明

### 插件结构

```
astrbot_plugin_stationsearcher/
├── main.py              # 插件主代码
├── metadata.yaml        # 插件元数据
├── requirements.txt     # 依赖列表
├── stations_database.json  # 默认数据库（初始数据）
└── README.md            # 说明文档
```

### 核心功能

- 使用 `@filter.command` 装饰器定义指令处理器
- 使用 JSON 文件存储和管理车站数据
- 支持异步操作，确保不阻塞主程序
- 完整的错误处理和日志记录
